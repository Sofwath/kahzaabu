How this works

kahzaabu is an automated archive and analysis of public press releases from presidency.gov.mv. It scrapes, indexes, and runs LLM-assisted analysis to surface claims and inconsistencies. The name "kahzaabu" means "falsehood" in Dhivehi.

1. Data collection

Every 12 hours, the pipeline checks presidency.gov.mv for new press releases, speeches, VP speeches, and news bulletins. New articles are added to a local SQLite archive. Both the English and Dhivehi versions are stored when available. Currently the archive covers articles from the Muizzu administration onwards (since 2023-11-17).

2. Claim extraction

For each new article, a language model (Claude Sonnet) extracts structured claims:

  • numeric_promise, deadline_promise — forward-looking commitments
  • numeric_update — current-status numbers (reserves, units delivered, etc.)
  • credit_claim — taking credit for delivery/completion
  • policy_assertion, boast, comparison_to_predecessor, denial

Each claim is stored with the verbatim quote it came from and a link back to the original article.

3. Fact-check curation

Weekly, the pipeline groups claims by topic and identifies inconsistencies across time:

  • LIE — a definite factual claim that is demonstrably false
  • CONTRADICTION — one claim directly contradicts another
  • MISLEADING — framing that misrepresents reality (e.g. metric switching)
  • SHIFTING NUMBERS — same subject given different values over time
  • CREDIT THEFT — claim of credit for previous-government project
  • BROKEN DEADLINE — stated deadline passed without delivery

4. Web verification

High-impact items (LIE, CONTRADICTION, SHIFTING NUMBERS, CREDIT THEFT) are sent through web search to cross-reference against independent reporting from Maldivian outlets (Mihaaru, Sun.mv, Edition.mv, AvasOnline, Maldives Independent) and international wires. All sources Claude actually retrieves are stored as evidence, labelled confirms, contradicts, context, or unclear.

5. Per-article inspection

Each article also gets a "fact card": a short summary, the most checkable claims, a history check against prior statements on the same topic, and a visualization (timeline or before/after) when applicable.

6. EN vs Dhivehi comparison

When both English and Dhivehi versions exist, the pipeline LLM-compares them and flags numeric_discrepancy, omission, softening, embellishment, or other substantive factual differences. Each flagged divergence shows the verbatim text from both languages and a literal back-translation of the Dhivehi.

7. Human review before publication

Items produced by the pipeline are not auto-published on this site. Each fact-check is reviewed by a human moderator who can publish it, reject it, or edit the public summary before it appears. Items with substantive evidence are prioritized.

Limitations & caveats

  • LLM outputs can be wrong; human review is the second line of defense, not the first.
  • "Lying" is a conclusion the reader draws. The categorisations here describe the evidence pattern (e.g. SHIFTING NUMBERS), not motive.
  • Translation differences may reflect editorial choice, not deception.
  • Press-release context can be missing — articles cite the original presidency.gov.mv URL so you can verify.
  • The archive is incomplete for periods before 2023-11-17.

Code & transparency

The full pipeline source — scraper, extractor, LLM prompts, web search verifier, translation comparator — is open-source and self-hostable. Every fact-check links back to the underlying article on presidency.gov.mv.
